A Systolic Array Implementation Using FPGAs

Using FPGAs to create systolic arrays can impact several aspects of Software Defined Radio including the parallel implementation of an adaptive beamformer.

Developments in microelectronics have revolutionized computer design: integrated circuit technology has increased the number and complexity of components that can fit on a chip. Consequently, this technology allows the building of low-cost, special-purpose, peripheral devices to rapidly solve sophisticated problems.

Very large scale integrated (VLSI) circuit technology makes it clear that simple and regular interconnections lead to inexpensive implementations and high densities, and high densities enable both high performance and low overhead. For these reasons, we are interested in designing parallel algorithms that have simple and regular data flows. We are also interested in using pipelining as a general method for implementing these algorithms in hardware. By pipelining, processing may proceed concurrently with input and output, and, consequently, overall execution time is minimized. Pipelining plus multiprocessing at each stage of a pipeline should lead to the best possible performance.

The Concept of a Systolic Array

One such implementation is the systolic array, composed of an array of separate processing cells. This array may be arranged in any regular structure. As a simple example of a rectangular systolic array, see Figure 1. Each cell has its own local memory and, as Figure 1 depicts, is connected to its nearest neighbor. Such a design allows waves of data (vectors or sampled wavefront) to be clocked through it in a highly regular occurrence analogous to that of a pumping heart; hence the name “systolic.”

A pair of systolic arrays may be used to solve the linear least-squares problem that arises in a wide range of signal processing situations. The main (triangular) array is used to implement a pipelined sequence of Givens rotations, which reduces the data matrix, X(n), to upper triangular form by unitary transformations (QR decomposition). Back substitution is then used to rapidly derive an optimal weight vector.

Using Systolic Arrays in Wireless Communications

There are a growing number of important real-time applications, especially in the area of wireless communications, in which systems are required to operate reliably in the presence of strong interference. Modern wireless communication systems based on the code division multiple access (CDMA) technology are limited in capacity and performance by three major factors: multipath fading, multiple access interference (MAI) and intersymbol interference (ISI). The commonly proposed schemes to overcome these impairments are the use of the transmitter power control, the error control coding and the diversity technique that is typically represented by the conventional rake receiver.

Performance of the rake, however, has been substantially degraded in the presence of a rapid time-varying channel, which is the usual case for a practical mobile radio system. There are two kinds of interference associated with a rake receiver for a CDMA downlink: one is the inter-finger interference (IFI); the other is MAI, both of which are due to the frequency selectivity of the wireless channel. IFI and MAI limit the CDMA system capacity when rake receivers are applied.

Suppression of IFI and MAI is needed to improve the performance of CDMA transmission. When the delay spread is large, the frequency selective channel may be transformed into a frequency non-selective fading channel through channel equalization. The equalization receiver based on adaptive algorithms seems to be an effective CDMA receiver for recovering the transmitted data through the restoration of the orthogonality of the spreading codes, thus suppressing both IFI and MAI.

Adaptive least-mean square (LMS) and recursive least-squares (RLS) calculate the time-varying channel iteratively and they have the advantage of shorter processing delay with relatively short buffer size for data storage. The RLS algorithm takes into account all the information that extends back to the initialization stage and updates the estimation of the tap-weight vector upon the arrival of new data. RLS is preferred over LMS due to its superior convergence properties. In addition, if the adaptive algorithm diverges, or converges slowly, it will be difficult to achieve the suppression of IFI and MAI, which is the very basic objective of an equalization receiver.

On the other hand, the RLS algorithm requires considerable operations in symbol duration, which is not desirable for a realistic radio system in which simplification is a crucial issue for cost reduction. The orthogonal triangularization of the input matrix via QR-decomposition (QRD) is introduced under such circumstances to improve the numerical property of the RLS filtering. Such a triangularization process can be realized through a series of Givens rotations, which is commonly employed to perform the QR updating on a sample-by-sample basis.

An efficient parallel triangular systolic processor array realization of the QRD-RLS algorithm using Givens rotations has been developed. A uniform cyclic clock controls the systolic array and it executes plane rotations to annihilate certain elements of the input signal matrix. Commonly, the computation of rotation angles requires inverse square roots, multiplications and additions. This is called the basic Givens rotations, but square-root-free Givens exist; also the CORDIC algorithm can be implemented that uses only binary shifts and adds, but at added cost due to required scaling, more iterations and possible unstable points in calculation. One thing that needs to be kept in mind is that when dealing with complex data, the rotations must first zero out the imaginary part followed by another rotation to zero out the remaining real part. Notice that the last column of cells resembles a single tap finger of the conventional rake receiver.

The systolic array of Figure 4 operates in a highly pipelined fashion. The computational wavefront propagates at the received symbol rate. For situations in which the desired signal, y(n), is unknown a priori, an estimate is used. In this case, a is fed back as input for y(n). Because of this feedback loop, full pipelining cannot be applied. However, the columns of the processing array can be pipelined, whereas rows cannot.

One important potential advantage of such a systolic array implementation is its energy efficiency. Energy consumption is of considerable importance in the development of interference suppression algorithms for wireless systems. Energy consumption in integrated circuits is an increasing function of gate speed. The large parallel systolic array processing allows the individual gate speed to be reduced significantly without a corresponding reduction in the updating speed of the algorithm, and thus achieves energy efficiency.

Good numerical performance is achieved by performing the algorithm using Givens rotations, which allows the use of reduced wordlength arithmetic for efficient implementation on both ASICs and FPGAs. Word length has to be sufficient to calculate the weights to an accuracy that allows full cancellation of the interference. Addressing FPGAs only, the architecture needs to be parameterized for wordlength and include attributes to provide placement information to achieve very dense layout with predictable timing.

The rate of improvement in FPGA technology is staggering. The latest FPGAs offer a means to meet the performance requirements of adaptive systems without adopting ASICs. Like DRAMs, they take full advantage of the massive number of transistors offered by current fabrication technology, and these can be exploited by suitable parallel implementations. In addition, FPGAs such as the LatticeECP family are now incorporating DSP blocks that perform hard multiplications, adds and MAC functions. Consequently, combining this technology improvement with architecture optimization is likely to yield capabilities far in excess of current architectures.

The Impact of Systolic Array Processors on Software Defined Radio

One area of technology in which the systolic array processor will have impact is Software Defined Radio (SDR). The SDR concept has been considered for some time, but the speed and cost of existing processors has prohibited commercial implementation. With the increasing capabilities of FPGAs and the requirements of multi-mode radio (i.e., multiple standards in one radio), SDR is emerging as an important technology for both military and commercial communications. The concept of reusing hardware blocks by reconfiguring the RF front end while also reconfiguring the baseband software is very appealing.

Parallel implementation of an adaptive beamformer is very well suited for realization on a configurable hardware platform. A systolic array may be used to process the data from a linear array of multiple, equally spaced omnidirectional sensors for beamforming. For the realization of a structural-/time-adaptive beamformer on a (re)configurable parallel hardware device, FPGAs would be used. Reconfiguration for different types of beamforming algorithms, e.g., MVDR and LCMVB, can be achieved by changing the processing algorithm within each of the cells and/or the array’s configuration. Even changes such as number of available sensors, number of users, number of constraints, number of interferers from known/unknown directions and number of required output signals can be accommodated for a particular beamforming algorithm.

Thus the combination of configurable hardware, systolic array structures, parallelism and pipelining will lead to a beneficial change in the way engineers think about implementing algorithms. Systolic array processing can satisfy many of the design requirements posed by communications receivers, enhancing system performance by allowing more matrix/vector processing on large amounts of data.
